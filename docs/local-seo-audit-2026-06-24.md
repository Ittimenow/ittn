Дата проверки: 24 июня 2026  
Проверяемый адрес: `http://127.0.0.1:4321/` после `npm run build` и `npm run start`  
Фреймворк: Astro static build  
Тип сайта: корпоративный сайт / услуги / кейсы / блог  
Production-домен: не указан; в сборке сейчас используется тестовый `https://ittimenow-ittn-27f0.twc1.net`

## 1. Сводное заключение

Сайт не готов к SEO-публикации на основном домене. Блокирующие причины: весь сайт закрыт от индексации сразу на трех уровнях (`robots.txt`, HTML meta robots, `_headers` с `X-Robots-Tag`), canonical всех 38 страниц указывает на тестовый домен, XML Sitemap отсутствует.

Положительные результаты локальной проверки: production-сборка проходит успешно, 38 HTML-страниц генерируются, главная возвращает `200`, случайный несуществующий URL возвращает `404`, битых внутренних HTML-ссылок и отсутствующих локальных ассетов в собранном `dist` не найдено.

## 2. Что проверено

- Сборка: `npm run build`.
- Локальный preview: `npm run start`, порт `4321`.
- HTTP: `/`, `/robots.txt`, `/sitemap.xml`, `/test-non-existent-page-12345/`, варианты URL со слешем, регистром и параметром.
- Статический обход `dist`: 38 HTML-страниц, 1132 внутренних ссылок, 326 изображений, 5 форм, JSON-LD.
- Поиск по коду: `robots`, `noindex`, `canonical`, тестовый домен, analytics/schema.

Ограничение: встроенный браузер в сессии недоступен, локальные зависимости `playwright` и `lighthouse` не установлены. Поэтому визуальная мобильная проверка, консоль браузера, гидратация и лабораторные Lighthouse/CWV требуют отдельного запуска.

## 9. Рекомендации разработчикам

1. Вынести SEO-запреты в env-флаг: staging закрыт, production открыт.
2. Заменить `site` в Astro config на production-домен перед релизом.
3. Добавить `@astrojs/sitemap` или собственную генерацию `/sitemap.xml`.
4. Настроить trailing slash и параметрические URL: единый canonical-формат и редиректы.
5. Исключить тестовые страницы из production build или закрывать их адресно.
6. Подключить рабочие формы и защиту от повторной отправки.
7. Оптимизировать тяжелые медиа, особенно видео и hero PNG.
8. Добавить schema на базовые шаблоны.

## 10. Рекомендации SEO-специалисту

1. Утвердить production-домен, www/non-www, trailing slash и список индексируемых шаблонов.
2. Решить судьбу `/b24new/`: отдельный лендинг или редирект/canonical на `/services/bitrix24/`.
3. Подготовить title/description шаблоны для услуг, кейсов, статей и служебных страниц.
4. Подготовить alt для значимых изображений.
5. Согласовать robots.txt, sitemap и правила для параметров.
6. Подготовить карту редиректов со старого сайта/старых URL, если миграция идет с существующего домена.
